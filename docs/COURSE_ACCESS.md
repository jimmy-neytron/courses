# Роли и доступ к курсам

Роль определяется отдельно для каждого курса:

- **Создатель** — пользователь из `courses.owner_id`. Он редактирует, публикует, приглашает и удаляет курс.
- **Ученик** — пользователь из `course_memberships`. Он видит опубликованный курс и проходит уроки, но не может открыть редактор или изменить материалы.

Глобальной роли «только автор» или «только ученик» нет. Один пользователь может создавать свои курсы и одновременно проходить чужие.

## Подключение к существующей базе

Откройте **Supabase → SQL Editor**, вставьте и выполните целиком:

`supabase/sql/10_course_access_roles.sql`

Миграция:

- создаёт `course_memberships` без таблиц аналитики и прогресса;
- создаёт безопасные коды приглашений;
- добавляет RPC для вступления по коду;
- обновляет RLS: ученик читает материалы, но изменять их может только создатель;
- создаёт коды для уже существующих курсов.

После выполнения SQL обновите приложение. Автор увидит кнопку «Пригласить», а пользователь сможет выбрать «Ввести код курса».
