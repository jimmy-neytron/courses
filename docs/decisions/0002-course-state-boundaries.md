# 0002. Границы состояния и persistence курсов

## Решение

- `stores/courses.ts` хранит реактивное состояние и координирует пользовательские сценарии.
- `services/course-repository.service.ts` инкапсулирует Supabase-запросы и преобразование загруженных данных.
- Узкие сервисы `course-access`, `course-cache`, `lesson-assets`, `lesson-audio` и `lesson-pdf` сохраняют ответственность за одну инфраструктурную возможность.
- Router принимает решения по готовому состоянию `auth` и не запрашивает сессию самостоятельно.

## LIFT

- **Locate**: имя файла прямо указывает, где искать persistence курсов.
- **Identify**: экспортируемые операции названы по действию и сущности (`createCourseRecord`, `saveCourseOrder`).
- **Flat**: сервис находится в существующем плоском каталоге `services`, без однофайловых уровней вложенности.
- **T-DRY**: Supabase-запросы не дублируются в store; локальная demo-логика остаётся рядом с состоянием, пока не появится второй потребитель.
