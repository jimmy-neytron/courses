# 0003. Универсальные профили курса без новых таблиц

## Решение

Существующая модель `courses → course_modules → lessons → lesson_blocks` используется для всех направлений обучения.

- Для языкового профиля `language_code` обозначает изучаемый язык.
- Для универсального профиля repository записывает техническое значение `und`, которое не попадает в доменную модель и интерфейс.
- Наличие `source_level` или `target_level` определяет языковой профиль курса.
- Пустые уровни определяют универсальный профиль.
- Шаблоны курса находятся в `src/data/course-templates.ts` и создаются только по явному выбору пользователя.
- Языковые и универсальные блоки используют одну таблицу `lesson_blocks`, но фильтруются в каталоге редактора.
- Набор секций урока определяется профилем курса и хранится в существующем versioned JSON-содержимом блока конфигурации.

Новые таблицы для типов курса, шаблонов и секций не создаются, пока у этих сущностей нет независимого жизненного цикла или второго источника данных.
