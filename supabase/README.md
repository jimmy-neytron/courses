# Supabase

## Новый проект

Выполните файлы в SQL Editor строго по порядку:

1. `sql/00_extensions_and_types.sql`
2. `sql/01_profiles.sql`
3. `sql/02_course_content.sql`
4. `sql/03_functions_and_triggers.sql`
5. `sql/04_rls.sql`
6. `sql/05_storage.sql` — необязательно, только для загрузки файлов

Затем создайте `.env` из `.env.example`.

## Уже существующая база

Сначала сделайте резервную копию. Затем выполните:

```text
sql/90_migrate_existing_to_courses_only.sql
sql/91_migrate_accent_to_green.sql
```

Первая миграция сохраняет курсы, модули, уроки, блоки и релизы. Она удаляет организации,
участников, приглашения, аудит, интеграции, API-ключи, webhooks и idempotency.

После миграции проверьте в приложении существующие курсы и только затем удаляйте
старые SQL-файлы из своей прежней папки. Вторая миграция устанавливает зелёный цвет по умолчанию и обновляет прежний стандартный цвет существующих курсов.
